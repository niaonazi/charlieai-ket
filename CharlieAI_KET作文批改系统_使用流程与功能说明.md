# CharlieAI KET作文批改系统：使用流程与功能说明

适用对象：KET / A2 Key 写作课程任课老师、主教、教务或需要批量整理作文反馈的老师。

这套系统不是替代老师判断，而是把“AI批改、结果录入、班级分析、家校反馈、主教复盘、Excel交付”连成一个稳定流程。老师仍然负责教学判断和最终复核，系统负责整理、提效和标准化交付。

## 一、系统能帮助老师做什么

1. 管理班级和学生名单。
2. 生成 KET / A2 Key 写作批改完整提示词。
3. 保存每位学生的最新批改结果。
4. 自动生成学生报告、家校反馈、老师复核建议。
5. 汇总班级均分、通过率、三维度能力数据和关注名单。
6. 一键导出 Excel 批改交付表。
7. 一键复制家校反馈和主教复盘。
8. 在课堂讲评页查看原文、修改、润色和总评。

## 二、使用前准备

老师需要准备：

1. Chrome 或 Edge 浏览器。
2. 一个可用的 AI 工具账号，例如豆包、ChatGPT 或其他能按提示词输出 JSON 的工具。
3. 学生名单。
4. KET 作文题目。
5. 学生作文原文。

重要提醒：

- 建议用 Chrome 或 Edge 打开系统。
- 如果要使用麦克风语音输入，建议通过本地网址打开，不要直接用 `file://` 打开。
- 学生数据默认保存在当前浏览器里。
- 更换电脑、清理浏览器缓存或换浏览器后，原数据可能看不到。
- 每次集中批改后，建议点击左下角“备份”保存数据。
- 给第三方 AI 工具发送学生作文时，建议使用学生缩写或编号，避免上传不必要的个人信息。

## 三、第一次使用流程

### 1. 打开系统

打开文件：

```text
KET作文批改系统.html
```

如果麦克风、复制或浏览器权限不稳定，建议通过本地服务地址打开，例如：

```text
http://127.0.0.1:8765/KET作文批改系统.html
```

### 2. 新建班级

点击左侧“新建”。

填写：

- 班级名称，例如：`KET周六上午班`
- 学生名单，可以用逗号或换行分隔

示例：

```text
陈一一，李明，王可，张小雨
```

或：

```text
陈一一
李明
王可
张小雨
```

点击“确定”后，学生会出现在左侧班级目录中。

## 四、单篇作文批改流程

### 1. 设置作文题目

点击左侧“AI批改”。

在“写作题目”里粘贴本次 KET 作文题目。

如果有班级要求，可以写在“补充评分标准或班级交付要求”里，例如：

```text
请重点检查是否回答所有 bullet points。家长反馈要简洁，不制造焦虑。
```

### 2. 复制完整提示词

点击“去豆包生成”。

系统会自动复制完整提示词，并跳转到豆包网页。也可以手动打开常用 AI 工具后粘贴。

新版提示词会要求 AI 输出完整中文报告，并在最后输出严格 JSON。JSON 里会包含三个完整范文版本：

- `Minimum Correction`
- `Score-Boost Version`
- `Examiner-Friendly Version`

### 3. 把学生作文发给 AI

在 AI 工具中粘贴系统生成的提示词，然后补充学生作文原文。

建议格式：

```text
学生姓名：陈一一
年级：五年级
学生作文：
Dear Tom,
Would you like come to my party?
Please bring some water.
See you soon,
Chen
```

### 4. 复制 AI 输出的 JSON

AI 通常会先输出中文报告，最后输出一个 JSON 代码块。

老师只需要复制最后的 JSON 代码块，通常长这样：

```json
{
  "score": 12,
  "recommended_score": 12,
  "conservative_score": 11,
  "final_teaching_score": 12,
  "words": 38,
  "content_score": 4,
  "organisation_score": 4,
  "struct_score": 4,
  "language_score": 4,
  "lang_score": 4,
  "bracket": "优秀",
  "summary": "任务基本完成，读者能轻松理解。",
  "parent_summary": "本次作文整体完成度较好。",
  "student_feedback": "继续注意动词形式。",
  "teacher_note": "下次可重点练习任务点完整回应。",
  "model_essays": {
    "Minimum Correction": "Dear Tom,...",
    "Score-Boost Version": "Dear Tom,...",
    "Examiner-Friendly Version": "Dear Tom,..."
  },
  "original_essay": "Dear Tom,..."
}
```

必须复制完整 `{ ... }`，不要只复制中文报告。

### 5. 录入系统

回到 CharlieAI 系统。

点击左侧对应学生姓名。

把 JSON 粘贴到弹窗里，点击“智能提取”。

系统会自动保存这位学生的批改结果，并显示在“阅卷”和“分析”页面。

## 五、阅卷页怎么用

“阅卷”页用于快速查看每位学生最新作文表现。

每张学生卡片会显示：

- 学生姓名
- 教学交付分
- 单题表现等级
- 总评摘要

点击学生卡片，可以打开完整报告。

报告包含：

- 教学交付分
- Recommended / Conservative 分数
- 单题表现等级
- Content / Language / Organisation 雷达
- 总评
- 学生反馈
- 家长总结
- 老师建议
- 语病修改
- 表达进阶
- 三个版本范文

其中三个范文版本分别用于：

- `Minimum Correction`：最低必要修改，适合保留学生原意。
- `Score-Boost Version`：A2 可模仿范围内的提分版本。
- `Examiner-Friendly Version`：更利于阅卷员判断任务完成度的自然版本。

## 六、分析页与交付中心

“分析”页现在更适合任课老师和主教做课后交付。

### 1. 交付中心

页面顶部是“交付中心”，包含四个核心按钮。

#### 导出 Excel

导出全班最新批改交付表，包含：

- 学生
- 状态
- 录入时间
- 教学交付分
- Recommended
- Conservative
- 等级
- Content
- Organisation
- Language
- 总评
- 家校反馈
- 学生反馈
- 老师建议
- `Minimum Correction` 范文
- `Score-Boost Version` 范文
- `Examiner-Friendly Version` 范文
- 原文

导出的文件为 `.xls`，可直接用 Excel 打开。

#### 复制家校反馈

一键复制全班家校反馈文本，适合发给家长、班群或教务。

反馈会按学生分段，包括：

- 本次 KET 写作单题表现
- 家校反馈
- 下一步建议

#### 导出反馈

导出 `.txt` 家校反馈文件，适合留档或二次编辑。

#### 主教复盘

一键复制班级层面的主教复盘，主要反映整个班级习作情况。

复盘内容包括：

- 班级习作概况
- 已批改人数
- 平均分
- 通过率
- 等级分布
- 需重点关注学生
- 共性特点
- 共性问题
- 后续教学建议
- 主训练模块和辅助训练模块
- 任课老师复核要点

### 2. 能力雷达

能力雷达显示班级三项能力均分：

- Content /5：任务完成
- Language /5：语言准确
- Organisation /5：组织结构

右侧会显示每一块的实际平均分、进度条和短评，例如：

- 优势项
- 基本稳定
- 需加强
- 优先补救

这部分适合主教判断后续课程重点。

### 3. 关注名单

关注名单可按两种方式筛选：

- 低于均分
- 低于通过线

点击学生姓名可直接打开报告。

## 七、指导页怎么用

“指导”页适合课堂讲评或小班面批。

使用流程：

1. 左侧选择学生。
2. 右侧查看原文、总评、修改和润色建议。
3. 点击“大屏投影”进入课堂展示模式。
4. 大屏里可切换：
   - 原文
   - 修改
   - 润色
   - 总评
5. 可用 A+ / A- 调整字体大小。
6. 可使用麦克风输入课堂临时记录。

课堂建议：

- 不要一次讲太多错误。
- 每位学生优先讲 1 个最影响分数的问题。
- KET 学生优先练“简单、准确、完整”，不要强行追求高阶表达。

## 八、备份与恢复

### 备份数据

点击左下角“备份”。

系统会下载一个 JSON 文件，文件名类似：

```text
CharlieAI_Backup_时间戳.json
```

建议每次集中批改后都备份一次。

### 恢复数据

点击左下角“恢复”图标。

选择之前备份的 JSON 文件。

恢复后，班级和学生记录会重新显示。

### 重置系统

点击左下角红色“重置”会清空当前浏览器里的所有系统数据。

除非确认已经备份，否则不建议点击。

## 九、常见问题

### 1. 麦克风点了没反应怎么办

建议用 Chrome 或 Edge 打开。

如果是直接双击 HTML，以 `file://` 打开的页面可能权限不稳定。建议通过本地地址打开，例如：

```text
http://127.0.0.1:8765/KET作文批改系统.html
```

同时检查浏览器地址栏左侧是否允许使用麦克风。

### 2. 范文板块为空白怎么办

新版系统已经兼容常见范文字段，并会显示 AI 返回的所有范文。

如果仍为空白，通常是 AI 输出的 JSON 里没有 `model_essays`，或三个范文字段为空。处理方式：

1. 使用新版完整提示词重新生成。
2. 要求 AI 重新输出严格 JSON。
3. 确认 JSON 中包含：

```json
"model_essays": {
  "Minimum Correction": "...",
  "Score-Boost Version": "...",
  "Examiner-Friendly Version": "..."
}
```

### 3. 系统提示“解析失败”怎么办

通常是 JSON 没有复制完整，或里面有多余文字。

处理方法：

1. 只复制 `{` 到 `}` 之间的内容。
2. 不要复制中文报告正文。
3. 如果仍失败，让 AI 重新输出严格 JSON。

可以对 AI 说：

```text
请只重新输出最后的严格 JSON 代码块，不要输出其他解释。
```

### 4. 为什么有 Recommended 和 Conservative 两个分数

`Recommended` 是正常阅卷推荐分。

`Conservative` 是考虑风险后的保守分，例如题目缺失、任务点不确定、作文信息不完整。

对家长沟通时，建议使用 `Final teaching score` 或系统里的“教学交付分”。

### 5. KET email 正文没有缩进要扣分吗

通常不扣。

KET email / message 不按中文作文格式要求首行缩进。`Dear Tom,` 后正文顶格写是可以接受的。

### 6. 学生只写名字，没有写 Love，要扣分吗

通常不扣。

KET email 落款不强制使用 `Love,`。可以建议学生使用：

```text
See you soon,
Write back soon,
Best wishes,
```

但不要把 `Love,` 当成必须格式。

### 7. 12/15 是不是最终考试优秀

不是。

12/15 只能说明这一篇写作单题表现较好。最终考试还要看完整 Reading & Writing、Listening、Speaking 的综合表现。

## 十、建议的日常使用节奏

### 小班 6-10 人

1. 课前收作文。
2. 用 AI 批改并录入系统。
3. 到“分析”页查看交付中心和能力雷达。
4. 导出 Excel 给内部留档。
5. 复制家校反馈发给家长或教务。
6. 复制主教复盘给主教看班级共性问题。
7. 课堂讲 1-2 个共性问题。
8. 指导页点名讲 2-3 个典型学生。
9. 课后点击“备份”保存数据。

### 一对一学生

1. 每次作文都录入系统。
2. 重点观察 Content、Organisation、Language 哪一项长期薄弱。
3. 用 Excel 留存阶段性记录。
4. 用家校反馈保持家长沟通稳定。
5. 下一节课只抓一个核心问题训练。

## 十一、老师使用原则

1. AI 结果不是官方成绩，老师保留复核权。
2. KET 作文不要按 PET、中高考、雅思或中文作文标准批改。
3. 小错不要夸大，先看读者能不能看懂。
4. 任务点没完成，语言再漂亮也不能高分。
5. 给家长反馈要稳定、清楚，不制造焦虑。
6. 给学生反馈最多抓 3 个重点，不要一次塞太多问题。

一句话总结：

CharlieAI KET作文批改系统适合把“批改、讲评、家校反馈、主教复盘、Excel留档”连成一个稳定流程。任课老师负责教学判断，系统负责整理、提效和标准化交付。
